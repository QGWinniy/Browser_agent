# Агент для работы с браузером

## Архитектура

### MCP-сервер  
В проекте реализован собственный MCP-сервер, который позволяет LLM выполнять команды в браузере, включая получение всех интерактивных элементов страницы.

### LLM-агенты  
В системе используются два взаимодействующих агента: **ManagerAgent** и **ExecutorAgent**.

- **ManagerAgent** анализирует текущее состояние страницы и предлагает несколько возможных действий для выполнения следующего шага.
- **ExecutorAgent** повторно анализирует страницу и выбирает наиболее подходящее действие из предложенных ManagerAgent’ом.

Такой подход повышает надёжность выполнения задач даже при использовании относительно слабых языковых моделей.

ManagerAgent также получает в промпте историю всех предыдущих действий, что позволяет ему учитывать допущенные ошибки и корректировать стратегию на следующих шагах.

Процесс завершается только тогда, когда ManagerAgent убедится, что цель достигнута.

## Тестирование

В демонстрационном видео видно, что агент успешно справился с поставленной задачей, 
несмотря на несколько допущенных ошибок. Вероятно, ошибки возникли из-за ограниченных возможностей модели или из-за большого объёма контекста (много элементов на странице). В
ажно отметить, что после каждой ошибки ManagerAgent смог скорректировать решение и в итоге привести к успешному выполнению задачи.
